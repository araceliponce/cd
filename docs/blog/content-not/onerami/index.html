Componentes:#

BreadCrumbs.#

Propiedad Descripción
items array de items, cada item tiene: icon, label(requerido), href, onClick (href y onclick definen si se renderiza un anchor o un button)
className agregar un className

Button.#

Elemento interactivo que dependiendo de sus atributos puede renderizarse como un botón o como un enlace.

Todo boton dentro de un form automaticamente recibe el poder de hacer submit, por lo que es important

Propiedad Descripción
variant Que determina las principales estilos que puede tener este botón (primary, secondary, danger)
href opcional. Si es un string, se renderizará como un enlace
disabled Deshabilita el elemento y obliga a que se renderice como button
as define como renderizar el elemento sin tomar en cuenta si usa las props href o disabled. seguirá teniendo el evento onClick aunque sea un ‘span’ o un ‘p’, etc
children contenido entre las etiquetas Button
innerRef en caso necesite hacerse alguna referencia al elemento
type acepta submit, reset o default
onClick evento en click
className con el fin de agregar estilos

Card.#

Propiedad Descripción
children Elementos contenidos dentro del card.
hasCloseButton Booleano. Permitiré incluir un botón que cierre el card.
onClose Permitirá agregar una función externa al momento de cerrar el card
isOpen Booleano. (predeterminado true) Para renderizar el card si es true
className En caso se necesita agregar un class name al Card
id En caso se necesite darle una id al card

Checkbox.#

Elemento que sirve para escoger una o varias opciones dentro de un mismo grupo

Propiedad Descripción
isChecked booleano. Puede ser establecido desde afuera del componente.
label Texto que se colocará en el label al lado del checkbox.
showsText Indica si se colocará o no la clase .sr-only al label
id id para el label y para el input de tipo checkbox.
name name para el input
onChange Devuelve el value del checkbox hacia afuera. Sirve para agregar alguna funcionalidad externa al elemento.
disabled Booleano. Para deshabilitar el checkbox
errors array de strings. no tiene prop hasErrorsOnTooltip
className agrega un classname

Chip.#

Propiedad Descripción
variant (predeterminado static). dynamic o static
options array de opciones, cada una con propiedades: value, label y color
value si es un chip estático, será un string, simple Pero si es dinámico, tiene que pertenecer A 1 de los ítems de options
color para chip estatico especificamente, es el background color del chip
onChange se ejecuta en cambio de opcion seleccionada y pasa el value seleccionado hacia afuera
hasTextInput (predeteminado false). agrega un form con un input y un btn submit
inputPlaceholder (predeterminado ‘’). define el placeholder del TextInput
onInputChange evento cuando el valor del input cambie
onInputSubmit evento cuando el valor del input se envie
className para agregar un classname

Input.#

Es un elemento interactivo que en su forma más básica se renderiza como un campo que el usuario llenará

Propiedad Descripción
type (predeterminado es text) text, tel, number, email, password, o cualquier string
id para establecer id
name para establecer name
label Generará el texto para el elemento label colocado por encima del input.
placeholder Al igual que el atributo en HTML. Es un texto grisaceo
maxLength Al igual que el atributo en HTML. Indica la mayor cantidad de caracteres que acepta el input.
disabled Al igual que el atributo en HTML. Deshabilita el poder usar este input.
value Con este Se establecerá el valor para el input desde afuera del componente.
onChange Cada vez que el input cambie de valor. Se podrá ejecutar una acción adicional definida exteriormente. Devuelve el value al exterior.
onBlur Cada vez que se se pierda el foco en el input, puede servir para validad el valor dejado y mostrar error si se deja vacio o se ingresó valor incorrecto
required Booleano
className con el fin de agregar estilos
regex de tipo RegExp
errors (predeterminado []). Es un arreglo de strings. Si un input contiene más de cero errores se tornará con un outline rojo.
hasErrorsOnList para mostrar los errores en una lista debajo del componente
tooltipEl puede colocar ’errors’ o un ReactNode. Si es un reactNode deberia usar un tooltip pero podria ser cualquier elemento
options solo en caso sea input type=tel, porque incluye un select y necesita pasarse el array de paises

Modal.#

El modal es un pequeño aviso que aparece por encima del contenido e inmoviliza el scroll en la pestaña hasta que el usuario tome la acción de aceptar, cancelar, continuar, etc.

Propiedad Descripción
isOpen Booleano. Indica si el modal está abierto, es decir, visible en la pestaña.
onClose Permitirá agregar una función extra a ejecutar después del cierre del modal.
closesOnOverlayClick Booleano (predeterminado: false). para escoger si el modal se cerrará al dar clic en el overlay. (Si es un modal poco importante, sería más fácil de cerrar, en cambio, si es un modal importante o con un form dentro deberia evitar que se cierre de forma tan sencilla)
size (preseterminado: default) Default o wide
title opcional. se renderiza en el mismo div que contiene al button
children ReactNode
footer acepta ReactNodes. por ejemplo footer= {<DangerModalButtons/>}
id En caso necesite colocarse un id al modal.
innerRef En caso necesites hacer referencia al modal.
className Para agregar una classname

RadioGroup.#

Elemento que sirve para escoger solo una opcion dentro de un mismo grupo de opciones. Es la misma funcion que la de un select

https://blog.prototypr.io/7-rules-of-using-radio-buttons-vs-drop-down-menus-fddf50d312d1 Usar en lugar de un select cuando:

  • hay pocas opciones: 2 , 3, 4, 5 o maximo 6.
  • le conviene al usuario ver las opciones de una vez (escoger entre 3 planes de tarifas)
Propiedad Descripción
value string. Valor que puede ser establecido desde afuera del componente.
name name que se colocará en cada input radio. –NO TIENE PROP ID
onChange Devuelve el value seleccionado hacia afuera. Sirve para agregar alguna funcionalidad externa al elemento.
disabled Booleano. Para deshabilitar el elemento
errors array de strings. no tiene prop hasErrorsOnTooltip
className agrega un classname

Select.#

Elemento compuesto de un select que contiene varias opciones. https://blog.prototypr.io/7-rules-of-using-radio-buttons-vs-drop-down-menus-fddf50d312d1 Usar cuando:

  • hay muchas opciones, mas de 6
  • hay alguna opcion predeterminada o recomendada

showsLabel agrega clase .sr-only al label
disabled Al igual que el atributo en HTML. Deshabilita el poder usar este input.
onChange Cada vez que el select cambie de valor. Se podrá ejecutar una acción adicional definida exteriormente. se envia el value seleccionado
required Booleano
className agrega un classname
errors (predeterminado []). Es un arreglo de strings. Si un input contiene más de cero errores se tornará con un outline rojo.
hasErrorsOnList para mostrar los errores en una lista debajo del componente
tooltipEl puede colocar ’errors’ o un ReactNode. Si es un reactNode deberia usar un tooltip pero podria ser cualquier elemento

Table.#

Elemento tabla que:

  • puede ordenarse al dar clic en los botones colocados en cada encabezado de cada columna de forma asc y desc.
  • cuenta con paginacion y select para cambiar n de rows
Propiedad Descripción
data Arreglo de filas. Cada fila es un objeto con varios pares de key: value.
columns Arreglo que indica las propiedades de cada columna (columna es vertical = 1 encabezado y varias celdas) .
Las props de las column es selector(string que identifica a esa columna), name (nombre visible en la tablas), grow: establece el min-width de la columna(por defecto las filas tienen un flex-basis de 100%, por lo que intentan ocupar el mismo ancho en la tabla, y un min-width de 7ch), align para alunear el texto/elementos dentro de las filas del body de esa columna, isvisible (si a una columna se le coloca isVisible: false esa columna no se renderizará), isSortable( si se le coloca false, el encabezado de esa columna no tendra flechas pequeñas y no podrá hacer cambios al orden de las filas), defaultSortOrder (‘asc’ o ‘desc’, la columna que tenga este key:value será la que mande en el ordenamiento inicial de las filas ), cellFormatter y headerFormatter (para reemplazar los contenidos de las celdas del tbody o del thead por algo más que solo texto)
rowsInPage Cantidad de filas por página. (predeterminado data.length)
innerRef Referencia que permitirá hacer referencia al elemento tabla, aunque este dentro de otros contenedores.
className Para agregar una classname

Crear una libreria de componentes con react#

https://dev.to/receter/how-to-create-a-react-component-library-using-vites-library-mode-4lma

para crear una libreria, desde cero, que pueda publicarse con npm publish o empaquetarse con npm pack:

npm create vite library –template react-ts

(eliges react y typescript normal)

dentro de src no creas nada.

creas una carpeta que puedes llamar ’lib’ (a la misma altura que src), y dentro una carpeta ‘components’.

dentro cada componente necesita su propia carpeta, la carpeta puede llamarse button y contener unicamente un index.ts que tenga todo el codigo del componente (opcionalmente un archivo .module.css que en el archivo index.ts importara si es que lo usa)

la carpeta ’lib’ tendrá dentro un main.ts que importe todos los componentes que quieras se compilen con npm run build

edita vite.config.ts

al final queda asi:



+ import { defineConfig } from 'vite'

+ import react from '@vitejs/plugin-react'

+ import { resolve } from 'path'

+ import dts from 'vite-plugin-dts'

+ import { libInjectCss } from 'vite-plugin-lib-inject-css'

+ export default defineConfig({

+ plugins: [

+ react(),

+ libInjectCss(),

+ dts({ include: ['lib'] })

+ ],

+

+ build: {

+ copyPublicDir: false,

+ lib: {

+ entry: resolve(__dirname, "lib/main.ts"),

+ name: "library",

+ fileName: "main", //esto decide el nombre de los archivos .js y .umd.cjs hijos de dist/

+ },

+ rollupOptions: {

+ external: ["react", "react-dom", "react/jsx-runtime"],

+ output: {

+ globals: {

+ react: "React",

+ "react-dom": "React-dom",

+ },

+ },

+

+ },

+ },

+ })

npm i @types/node -D (para usar path)

edita el tsconfig.json para aceptar tambien de la carpeta lib

"include": ["src", "lib"],

crea una un nuevo tsconfig-build.json (u otro nombre), Éste se usará para un build que evite tomar en cuenta la carpeta src.

{

"extends": "./tsconfig.json",

"include": ["lib"]

}

reemplaza el script build de package.json por

"build": "tsc --p ./tsconfig-build.json && vite build",

copia el archivo vite-env.d.ts de src y pegalo dentro de ’lib’

((para que vite.svg y otros archivos estaticos dentro de la carpeta public se exporten también en el dist), en vite.config.ts puedes agregar dentro de build: {

+ copyPublicDir: false,

...

}

para que se cree el build con las definiciones de tipos (archivo main.d.ts):

npm i vite-plugin-dts -D

y:

+import dts from 'vite-plugin-dts'

…

plugins: [

react(),

+ dts({ include: ['lib'] })

],

…

instala los paquetes necesarios y configuraciones para evitar errores por usar any o tener imports no usados:

npm i styled-components

npm install --save-dev @iconify/react

y agregué al eslint:

"@typescript-eslint/no-explicit-any": ["off"],

"no-unused-vars": "off",

"@typescript-eslint/no-unused-vars": "error"

con 2 componentes, library.js pesa 107kb, porque react y react/jsx-runtime se estan compilando tambien

en vite.config.ts :

external: ["react", "react-dom", "react/jsx-runtime"],

ahora pesa 85kb

notas:

para exportar solo acepta que en la misma linea diga export. por ejemplo export function Input o export const Input. marca error si colocas al final de todo el codigo export Input

Vita utiliza/soporta modulos css predeterminadamente. solo necesita crearse un nombre.module.css en cualquier carpeta de componente y luego importarlo en el tsx del componente

https://vitejs.dev/guide/features#css-modules

Pero los estilos de todos tus componentes se van a juntar en una sola hoja de estilos CSS. El problema es que si quieren usarlo van a tener que importarlo. Por lo cual tenemos que instalar un plugin que va a inyectar el CSS en su componente correspondiente.

npm i vite-plugin-lib-inject-css -D

y agregarlo en el vite.config.ts

+ libInjectCss(),
+

Con esto ya no es necesario importar esa hoja css en el proyecto cada vez

pero aun asi los estilos de todos los module.css se van a juntan en un mismo main.css e importar a la vez, aunque solo uses por ejemplo un estilo de pocas lineas, importarías todo. por eso

npm i glob -D

(glob no funcionó bien, asi que lo reverti hasta usar solo injectcss)

Luego ya se puede pasar a publicar, como el package.json va a ser publicado, verificar detalles como el name @nombre-usuario-npm/nombre-a-escoger y private debes cambiarse a false, y agregar

+ "main": "dist/main.js", // deben ser los mismos que estan dentro del dist/

+ "types": "dist/main.d.ts", //comprueba si colocaste un filename distinto en vite.config.ts

+ "files": [

+ "dist"

+ ],

y colocar:

"react": "^18.2.0",

"react-dom": "^18.2.0",

"styled-components": "^6.1.8"

como peerDependecies y también incluirlos dentro de devDependencies

Para garantizar que se haga el build antes de publicar el package, en package.json

+ "prepublishOnly": "npm run build"
